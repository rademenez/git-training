GIT

Gentle intro

Created by Rademenez

What for?

  • distributed, non-linear, large-scale project
  • "everything from small to very large projects with speed and efficiency"
  • GitHub

Features

  • easy non-linear development
  • any workflow
  • fast and secure
  • implicit backup
  • de-facto standard
  • free and open-source

Hurdles

  • powerful and complex
  • cryptic command-line interface
  • late integration

Concepts

  • commit is a changeset with SHA1 hash id
  • commits make Directed Acyclic Graph
  • branches are pointers
  • HEAD is the current branch
  • working dir, staging, repository
  • many repositories

Magic spells

  • init [--bare], clone
  • status, diff, diff --cached, log [--graph]
  • add [-p], reset
  • commit
  • pull, fetch, push [--force]
  • checkout
  • merge [--ff], rebase, rebase -i, cherrypick

Workshop

  • Dev on master, release branches
  • John: README
  • Jane: first impl
  • John: exception for -1 (merge/rebase)
  • Jane: comment
  • John: copyrights
  • Jane: Release 1.0
  • John: Bug fix to 1.0 and master
  • Jane: Perf improv
  • John: Add comment
  • Jane: Format code (rebase -i)

  • kdiff, meld, cdiff, BeyondCompare, P4Merge, Kaleidoscope, diffmerge
  • gitignore.io
  • GitHub, GitLab, Stash
